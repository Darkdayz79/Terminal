🔐 Understanding File Permissions

Every file and folder on a Unix system has a set of permissions that control who can read it, write to it, or execute it. Once you can read a permission string, you can control exactly who has access to anything.

Reading the Permission String

Run ls -la and you'll see a column like -rwxr-xr-x at the start of each line. This 10-character string tells you everything about who can do what with that file.

The Three Permission Groups

Permissions are always defined for three groups of people, in this order:

U
Owner (User)
The person who created the file. On your Mac, this is usually your account. The owner gets the first 3 permission characters after the file type. The owner can always change a file's permissions with chmod.
G
Group
A named collection of users. On macOS, most files belong to the staff group. Groups let you share access with a team without opening it to everyone. The group gets characters 4–6.
O
Others (World)
Everyone else — any user on the system who isn't the owner and isn't in the group. These are the last 3 characters. Web servers often use "others" permissions to serve public files.

Octal Notation — The Numbers Behind Permissions

The chmod command accepts a 3-digit number like 755. Each digit represents the permissions for Owner, Group, and Others respectively. Each digit is the sum of: 4 (read) + 2 (write) + 1 (execute).

⚠ About sudo and root
The root user (superuser) ignores all permission restrictions — root can read, write, and execute any file regardless of its permissions. When you use sudo, you temporarily act as root. This is why sudo requires a password and should be used carefully.

Changing Permissions with chmod

# Octal notation (recommended)
chmod 755 script.sh          # rwxr-xr-x — owner full, group/others read+exec
chmod 644 index.html         # rw-r--r-- — owner read/write, others read-only
chmod 600 ~/.ssh/id_rsa      # rw------- — private key, owner only
chmod -R 755 ./public/       # -R applies recursively to all files in folder

# Symbolic notation (easier to remember)
chmod +x script.sh           # Add execute for everyone
chmod u+x script.sh          # Add execute for owner only
chmod go-w file.txt          # Remove write from group and others
chmod a=r file.txt           # Set all (a) to read-only

Changing Ownership with chown

Permissions tell you what can be done. Ownership tells you who the file belongs to. Use chown to transfer ownership.

# Change owner only
sudo chown darkness file.txt

# Change owner and group
sudo chown darkness:staff file.txt

# Change group only
sudo chown :developers project/

# Recursively change ownership of a whole directory
sudo chown -R darkness:staff /var/www/mysite/
💡 Tip — Why SSH keys stop working
SSH private keys must have exactly 600 permissions (-rw-------). If the file is too permissive (e.g. 644), SSH refuses to use it as a security measure. Fix it with chmod 600 ~/.ssh/id_rsa.

📁 File Operations

The terminal gives you precise, scriptable control over every file on your system. These are the core operations you'll use daily — creating, reading, editing, copying, moving, and deleting files and folders.

Creating Files and Folders

touch
Create empty file
Creates a new empty file instantly. If the file exists, it just updates the timestamp.
touch notes.txt
mkdir
Create folder
Creates a new empty directory. Add -p to create nested folders in one command.
mkdir -p src/components/ui
echo >
Create file with content
Creates a file and writes text into it in one step using output redirection.
echo "Hello" > welcome.txt
cat >>
Append to file
Appends text to the end of an existing file without overwriting it (>> vs >).
echo "new line" >> log.txt
💡 Redirection: > vs >>
> overwrites the file completely. >> appends to the end. Always double-check which one you're using — overwriting the wrong file with > is a common mistake.

Reading Files

# Print entire file to terminal
cat notes.txt

# Read large files page-by-page (space=next page, q=quit)
less /var/log/system.log

# Show just the first 10 lines
head access.log

# Show just the last 10 lines
tail access.log

# Watch a log file live as new lines are added
tail -f /var/log/system.log

# Search inside a file for a word
grep "error" app.log

Editing Files

There are several ways to edit a file directly from the terminal.

1
nano — the beginner-friendly editor
Opens the file in a simple editor right in your terminal. Control shortcuts are shown at the bottom. ^X = Ctrl+X to exit, ^S = save. No modes to learn.
nano notes.txt
2
vim — the power editor
Opens in Normal mode. Press i to enter Insert (typing) mode. Press Esc to return to Normal mode. Type :wq to save and quit, :q! to quit without saving.
vim server.js
3
open -e — opens in TextEdit
Launches the file in macOS TextEdit — a GUI app, but triggered from the terminal. Useful when you want a graphical editor for quick edits.
open -e config.txt
4
sed — non-interactive find and replace
Replace text in a file without opening it. The -i flag edits the file in-place. Great for scripting bulk changes.
sed -i '' 's/localhost/prod.server.com/g' config.txt

Copying Files and Folders

# Copy a single file
cp report.pdf ~/Desktop/report-copy.pdf

# Copy into a folder (keeps the original filename)
cp report.pdf ~/Desktop/

# Copy an entire folder and its contents (-r = recursive)
cp -r project/ ~/Desktop/project-backup/

# Copy multiple files into a folder
cp *.txt ~/Documents/text-files/

# Verbose: print each file as it copies
cp -rv src/ backup/
⚠ cp overwrites without asking
If the destination file already exists, cp will silently overwrite it. Use cp -n (no-clobber) to skip existing files, or cp -i to be prompted before overwriting.

Moving and Renaming

mv does both moving and renaming — they're the same operation. If source and destination are in the same folder, it's a rename. If they're different paths, it's a move.

# Rename a file (same directory)
mv draft.txt final-report.txt

# Move a file to a different folder
mv invoice.pdf ~/Documents/Finance/2026/

# Move multiple files at once
mv *.log ~/logs/archive/

# Rename a folder
mv old-project/ new-project-name/

# Safe move: prompt before overwriting
mv -i source.txt destination.txt

Opening Files and Applications

# Open a file with its default app (like double-clicking)
open photo.jpg              # Opens in Preview
open index.html             # Opens in your default browser
open spreadsheet.xlsx       # Opens in Numbers or Excel

# Open the current directory in Finder
open .

# Open with a specific application
open -a "Visual Studio Code" .
open -a "Google Chrome" index.html
open -a "Preview" document.pdf

# Open a URL in the default browser
open https://github.com

# Open TextEdit for quick editing
open -e notes.txt

# Reveal a file in Finder without opening it
open -R ~/Documents/report.pdf

Deleting Files and Folders

⛔ Deleted files do NOT go to Trash
rm permanently deletes files with no recovery option. There is no undo. Always double-check your command before pressing Enter, especially when using wildcards or -rf. 
# Delete a single file
rm old-notes.txt

# Delete with confirmation prompt (safer)
rm -i important.txt

# Delete an empty folder
rmdir empty-folder/

# Delete a folder and ALL its contents (use with extreme care)
rm -rf old-project/

# Move to Trash instead of permanently deleting (macOS)
trash file.txt              # Requires: brew install trash

Working with Archives

# Create a compressed .tar.gz archive
tar -czf backup.tar.gz projects/

# Extract a .tar.gz archive
tar -xzf backup.tar.gz

# Extract to a specific folder
tar -xzf backup.tar.gz -C ~/Desktop/

# Create a .zip file
zip -r archive.zip folder/

# Extract a .zip file
unzip archive.zip

# List contents without extracting
unzip -l archive.zip

Finding Files

# Find by name in current directory
find . -name "*.pdf"

# Find files modified in the last 7 days
find ~/Documents -mtime -7

# Find and delete all .DS_Store files
find . -name ".DS_Store" -delete

# Spotlight search (searches file content too)
mdfind "quarterly report" -onlyin ~/Documents

# Find files larger than 100MB
find ~ -size +100M -type f

Every action you normally do by clicking and dragging has a terminal equivalent. This guide maps common macOS tasks to their command-line counterparts — giving you faster, automatable, and scriptable control over your entire computer.

Launching and Quitting Applications

open -a "Google Chrome"
open -a "System Preferences"

# Open an app with a specific file
open -a "Xcode" MyProject.xcodeproj

# Quit an app by name (graceful, like Cmd+Q)
osascript -e 'quit app "Slack"'

# Force quit a frozen app by name
killall Finder
killall "Google Chrome"

# Force kill by PID (use 'ps aux' or 'top' to find the PID)
kill -9 4821

# Restart Finder and Dock
killall Finder
killall Dock

Managing System Settings via Terminal

The defaults command reads and writes macOS system and app preferences — including many hidden settings not exposed in System Settings.

# Show hidden files in Finder
defaults write com.apple.finder AppleShowAllFiles YES
killall Finder   # restart Finder to apply

# Show the full file path in the Finder title bar
defaults write com.apple.finder _FXShowPosixPathInTitle -bool true

# Disable the "Are you sure you want to open this?" dialog
defaults write com.apple.LaunchServices LSQuarantine -bool false

# Speed up Dock animations
defaults write com.apple.dock autohide-time-modifier -float 0.15
killall Dock

# Show all file extensions in Finder
defaults write NSGlobalDomain AppleShowAllExtensions -bool true

# Disable auto-correct
defaults write NSGlobalDomain NSAutomaticSpellingCorrectionEnabled -bool false

# Read a current setting value
defaults read com.apple.finder AppleShowAllFiles

# Reset a setting to default
defaults delete com.apple.finder AppleShowAllFiles

Network Management

# List all network services
networksetup -listallnetworkservices

# Get current Wi-Fi network name
networksetup -getairportnetwork en0

# Connect to a Wi-Fi network
networksetup -setairportnetwork en0 "NetworkName" "password"

# Turn Wi-Fi off and on
networksetup -setairportpower en0 off
networksetup -setairportpower en0 on

# Set DNS servers (e.g. Cloudflare)
sudo networksetup -setdnsservers Wi-Fi 1.1.1.1 1.0.0.1

# Flush DNS cache
sudo dscacheutil -flushcache

# Get your public IP address
curl -s https://api.ipify.org

Automation with Scripts

The real power of the terminal is combining commands to automate tasks. Save any sequence of commands in a .sh file and run it whenever you need.

#!/bin/zsh
# save as cleanup.sh, then: chmod +x cleanup.sh && ./cleanup.sh

# Automated cleanup script
echo "Starting cleanup..."

# Empty Downloads folder older than 30 days
find ~/Downloads -mtime +30 -type f -delete

# Remove .DS_Store files everywhere
find ~ -name ".DS_Store" -delete

# Empty Trash
rm -rf ~/.Trash/*

# Show disk space remaining
df -h / | tail -1

say "Cleanup complete"
echo "Done."
💡 Making a script executable and running it
After writing a script, you need to give it execute permission before running it:
chmod +x myscript.sh — grants execute permission
./myscript.sh — runs it
Add it to ~/bin/ (and add that to your PATH) to run it from anywhere by just typing its name.
